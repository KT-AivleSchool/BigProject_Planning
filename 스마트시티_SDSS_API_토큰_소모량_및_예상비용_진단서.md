# [진단서] 플랫폼 시뮬레이션 가동 시 API 토큰 소모량 및 예상 비용 진단서 (API Cost & Token Analysis)

본 진단서는 플랫폼의 **1) AI 에이전트 데이터 감리 및 적재(ETL)** 단계와 **2) LangGraph Multi-Agent RAG 심의 토론** 단계가 1회 가동될 때 소모될 예상 API 토큰량과 이에 따른 정량적 운영 비용을 OpenAI 최신 모델 단가를 기준으로 편향 없이 진단한 기술 분석 보고서입니다.

---

## 1. 📊 플랫폼 가동 파이프라인별 토큰 소모량 예측 (Token Estimation)

### 1단계: AI 에이전트 기반 데이터 적재 (AI Ingestion Phase)
*   **작동 Rationale:** 파일 전체가 아닌 첫 5행의 스니펫(Snippet)과 헤더 정보만 발췌하여 입력하는 **하이브리드 스니펫 아키텍처** 적용 시의 토큰 산정입니다.
*   **소모 토큰량 (파일 1개 업로드당):**
    *   *Input (프롬프트 + 첫 5행 스니펫):* **약 1,500 토큰**
    *   *Output (정합성 판정 & 매핑 JSON):* **약 500 토큰**
    *   *1회 파일 적재 시 소모량:* **총 2,000 토큰 / 파일**

---

### 2단계: LangGraph Multi-Agent RAG 심의 토론 (Deliberation Phase)
*   **작동 Rationale:** 3인의 에이전트(보건관, 상인대표, 시민대표)가 용산구 금연 조례 RAG 데이터를 참조하여 후보지 속성(유동인구, 아동인구비율)을 읽고 2~3회 왕복 토론(총 6~9회 발언)을 수행하는 흐름입니다.
*   **소모 토큰량 (후보지 1개소 심의당):**
    *   *Input (프롬프트 + 조례 RAG 문맥 + 대화 이력 3회 누적):* **약 7,500 토큰** (RAG 컨텍스트가 주된 점유)
    *   *Output (에이전트별 의견 발언 총합 - 약 1,200자):* **약 1,500 토큰**
    *   *1개 후보지 심의 시 소모량:* **총 9,000 토큰 / 후보지**

---

## 2. 💸 모델별 API 예상 단가 대입 및 비용 분석 (Cost Projection)

OpenAI 공식 가격(1백만 토큰당 단가, $1 = 1,380원 기준)을 대입하여 환산한 1회 시뮬레이션당 실질 비용입니다.

| OpenAI 모델명 | Input 단가 (1M 토큰당) | Output 단가 (1M 토큰당) | 1회 시뮬레이션 총 비용 (달러) | 1회 시뮬레이션 총 비용 (원화) |
| :--- | :---: | :---: | :---: | :---: |
| **GPT-4o** *(최고 성능)* | $5.00 | $15.00 | **$0.0650** | **약 89.70원** |
| **gpt-4o-mini** *(가성비/권장)* | $0.15 | $0.60 | **$0.0020** | **약 2.76원** |

### 🧮 1회 시뮬레이션 패키지 총합 계산 예시 (PoC 흡연부스 기준)
*   *조건:* 데이터셋 파일 5개 일괄 적재 + 최종 추천 후보지 1개소에 대한 3인 AI 토론 및 PDF 컴파일 가동.
*   **총 소모 토큰:**
    *   데이터 적재 (5개 파일 \times 2,000토큰) = **10,000 토큰**
    *   AI 심의 토론 (후보지 1개소) = **9,000 토큰**
    *   *총합 소모 토큰:* **`19,000 토큰`** (Input: 16,000 / Output: 3,000)
*   **실질 비용 결과:**
    *   **GPT-4o 가동 시:** $0.080 (Input) + $0.045 (Output) = **$0.125 (약 172.50원)**
    *   **gpt-4o-mini 가동 시:** $0.0024 (Input) + $0.0018 (Output) = **$0.0042 (약 5.79원)**

## 🏁 3. 기술적 권고 사항 (Technical Recommendations)

1.  **개발 및 프로토타입 단계 (gpt-4o-mini 고정):**
    *   2주차 DB 적재 테스트 및 5주차 LangGraph 토론 로직 디버깅 중에는 개발 서버 API 키를 무조건 **`gpt-4o-mini`로 강제 고정**하여, 개발진의 실수로 인한 API 비용 폭탄을 방이함 (1만 번 시뮬레이션 시 단 5만 원 선에서 해결 가능).
2.  **공식 보고서 출력 단계 (GPT-4o 하이브리드 가동):**
    *   공무원이 최종 심의서를 PDF로 확정 인쇄하는 프로덕션 API 호출에 한해서만 최고 수준의 문장 윤문 및 조례 정합성 판단력을 보장하는 **`gpt-4o` 모델로 백엔드에서 일시 스위칭**하도록 설계할 것을 권장함.
3.  **RAG 캐싱 (Context Caching) 기술 적용:**
    *   금연 조례 등 고정된 대용량 텍스트는 OpenAI의 **'프롬프트 캐싱(Prompt Caching)'**이 자동으로 적용되므로, 실제 서비스 시 Input 토큰 비용의 **50%가 자동 할인**되어 상기 계산된 예상 비용보다 훨씬 더 저렴하게 가동될 것입니다.
