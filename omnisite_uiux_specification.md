# 📱 [제출용] OmniSite 플랫폼 UI/UX 상세 설계서 (v1.0.0-prototype)

본 문서는 **"9기 빅프로젝트 산출물 워크시트(AI트랙)"** 19페이지에 수록해야 하는 **UI/UX 설계서**의 공식 제출용 텍스트 명세 및 와이어프레임 가이드라인입니다.

---

## 🖥️ [화면 1] 데이터 일괄 업로드 및 AI 감리 대시보드

### 1. 화면 메타데이터
* **Project명:** 지능형 다목적 스마트시티 입지 선정 및 공공갈등 예측 플랫폼 "OmniSite"
* **화면 ID:** OMNI-USR-001
* **서버:** WEB (Next.js Frontend) / WAS (FastAPI Backend)
* **화면경로:** `홈 > 프로젝트 생성 > 데이터 업로드`
* **화면명:** 데이터셋 일괄 수신 및 AI 시맨틱 감리 대시보드
* **버전:** 1.0

### 2. 화면 구성 (Wireframe Sketch)
```text
========================================================================================
 [OmniSite]  Dashboard   Maps   Data   Models   Settings                 Admin. J. Park
----------------------------------------------------------------------------------------
 [ OMNI-USR-001 : 데이터셋 일괄 업로드 및 AI 감리 ]
 
  +-----------------------------------------------------------------------------------+
  |                                                                                   |
  |                        [ 📂 Drag & Drop Files Here ]                              |
  |             (공간 SHP/CSV, 일반 통계 CSV, 규제 조례 PDF/HWP 일괄 수신)             |
  |                                                                                   |
  +-----------------------------------------------------------------------------------+
 
  +-----------------------------------------------------------------------------------+
  | [ AI 시맨틱 감리 및 물리 라우팅 상태 ]                                            |
  |-----------------------------------------------------------------------------------|
  |  No | 파일명              | 용도 판별 (AI 추천)    | 상태          | 액션          |
  |  1  | bus_stops_ys.csv    | 대중교통 거점 (공간)   | [정상]         | [매핑 확인]   |
  |  2  | school_zone_ys.csv  | 교육 보호구역 (공간)   | [경고: 결측치] | [HITL 수정]   | [1-1]
  |  3  | complaint_log.csv   | 민원 건수 (정량)      | [정상]         | [매핑 확인]   |
  |  4  | smoking_law.pdf     | RAG 규제 조례 (문서)   | [텍스트 파싱] | [RAG 적재]    |
  +-----------------------------------------------------------------------------------+
 
                      [ < 이전단계 ]                 [ 2단계: 공간 검증 진행 > ]      [1-2]
========================================================================================
```

### 3. 화면 설명 (Interaction Flow)
* **[1-1] AI 감리 피드백 및 HITL 트리거:**
  * AI가 업로드된 CSV의 첫 5행 스니펫을 감리하여 공간 좌표계 누락, 주소 결측치 등을 감지합니다.
  * 정합성 경고가 뜬 행의 `[HITL 수정]` 버튼 클릭 시, 해당 결측 데이터를 수동 보완하거나 지오코딩 API를 강제 재가동하는 레이어 팝업이 출력됩니다.
* **[1-2] 프로세스 전이 버튼:**
  * 모든 필수 데이터셋의 감리 상태가 `[정상]` 또는 `[HITL 승인]` 상태로 완료되면 `[2단계: 공간 검증 진행]` 버튼이 활성화되어 공간 맵 화면(OMNI-USR-002)으로 전환됩니다.

### 3. 화면 설명 (Interaction Flow)
* **[2-1] 비주얼 HITL 및 공간 제한 경고:**
  * PostGIS `ST_Buffer` 연산으로 병합된 **적색 반투명 폴리곤 (Exclusion Mask)**을 Mapbox 지도 상에 오버레이 렌더링합니다.
  * 지오코딩 오차가 있는 시설물 핀을 마우스 드래그로 이동시킬 수 있으며, 이동 중 제한 마스크와 충돌 시 우측 알림창에 **`[!] 경계 침해 감지`** 경고가 실시간 출력되어 안전 입지 조정을 보조합니다.
* **[2-2] 프로세스 전이 버튼:**
  * 입지 보정이 완료되어 `[보정 위치 확정 등록]`이 완료되면 AHP 연산 화면(OMNI-USR-003)으로 이동합니다.

## 🖥️ [화면 3] AHP 가중치 잠금 및 AI 실시간 토론 SSE 중계

### 1. 화면 메타데이터
* **Project명:** 지능형 다목적 스마트시티 입지 선정 및 공공갈등 예측 플랫폼 "OmniSite"
* **화면 ID:** OMNI-USR-003
* **서버:** WEB (Next.js Frontend) / WAS (FastAPI Backend)
* **화면경로:** `홈 > 프로젝트 생성 > 입지 분석 및 갈등 예측`
* **화면명:** AHP 일관성 비율 잠금 및 LangGraph 실시간 SSE 토론 시뮬레이터
* **버전:** 1.0

### 2. 화면 구성 (Wireframe Sketch)
```text
========================================================================================
 [OmniSite]  Dashboard   Maps   Data   Models   Settings                 Admin. J. Park
----------------------------------------------------------------------------------------
 [ OMNI-USR-003 : 입지 평가 및 실시간 갈등 시뮬레이션 ]
 
  +------------------------------------------------+ +--------------------------------+
  | [ AHP 쌍대비교 슬라이더 ]                      | | [ AI 찬반 대리인 모의 토론창 ] |
  |------------------------------------------------| |--------------------------------|
  |  기준 A vs 기준 B  [---o---] 1~9               | | [시민대표 Agent]:              |
  |  기준 B vs 기준 C  [-o-----]                   | | "해당 부스는 스쿨존 경계에     |
  |                                                | |  너무 인접해 있어 청소년 간접  |
  |  - 일관성 비율 (C.R.): 0.07 [통과/녹색]        | |  흡연 위험도가 상승합니다."    | [3-1]
  |  [🔒 AHP 가중치 모델 잠금 (Lock) ]             | |                                |
  |                                                | | [상인대표 Agent]:              |
  |  +------------------------------------------+  | | "이태원 상가 활성화를 위해     |
  |  | [ Top 1 부지 ]  [ Top 2 ]  [ Top 3 ]     |  | |  가장 유동인구가 많은 1번     |
  |  |  - 용산구 한강로3가 40-1 (매칭 87점)         |  | |  위치가 상권에 적합합니다."    |
  |  |  - 갈등 민감도 벡터: [상]                     |  | |                                |
  |  |  [ ▶ 모의 토론 시뮬레이션 시작 ]             |  | | [!] SSE 스트리밍 토론 진행 중.. |
  |  +------------------------------------------+  | +--------------------------------+
  +------------------------------------------------+
 
        [ < 이전단계 ]       [ 📄 타당성 행정 결재서 PDF 인쇄 (WeasyPrint) ]          [3-2]
========================================================================================
```

### 3. 화면 설명 (Interaction Flow)
* **[3-1] AHP Lock 및 SSE 스트리밍 토론:**
  * 공무원이 쌍대비교 슬라이더를 움직이면 실시간으로 일관성 비율(C.R.)을 연산하여 보여줍니다. C.R. < 0.1 검증 만족 시 `[AHP 가중치 모델 잠금 (Lock)]` 버튼을 활성화하여 DB에 가중치를 동결 적재시킵니다.
  * 입지 스코어링이 완료되면 Top 1~3 후보지가 탭 구조로 제공되며, Top 1 탭 진입 시 LangGraph 토론 API가 자동 기동됩니다.
  * FastAPI에서 보내주는 한 글자씩 스트리밍 데이터(SSE)를 실시간 메신저 UI 형태로 한 자씩 타이핑 애니메이션으로 출력합니다.
* **[3-2] 행정 보고서 출력:**
  * 토론 합의가 마감되면 `[📄 타당성 행정 결재서 PDF 인쇄]` 버튼이 활성화되며, WeasyPrint가 백엔드에서 HTML-to-PDF 컴파일을 가동해 정식 행정 리포트를 파일로 내보냅니다.
