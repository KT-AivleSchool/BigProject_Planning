# 🏙️ 빅프로젝트 요약: OmniSite 플랫폼 (v1.0.0-prototype)

본 문서는 **"보건소 소관의 다목적 스마트시티 공간 의사결정시스템 (OmniSite)"**의 최종 완성본 기획 명세, WBS 주니어 R&R, 5단계 E2E 파이프라인 및 EKS 인프라 보안 구조를 포함한 옵시디언 노트입니다.

(연관 노트: [[빅프로젝트]])

---

## 1. 과제 개요 및 필요성
* **과제명**: 지능형 다목적 스마트시티 입지 선정 및 공공갈등 예측 플랫폼 (OmniSite)
* **목적**: 
  * B2G 행정 의사결정 시 부서 간 칸막이 병목을 제거하고, 공간정보(GIS) 분석과 LLM 다자 토론을 융합해 최적 입지 선정 및 타당성 결재 문서를 즉시 추출하는 행정 보조 도구 구축.
  * 기성 부스(개당 1,500만 원 상당) 설치 후 위법 시공 지적 및 주민 갈등으로 인한 강제 철거/이전(세금 낭비) 예산 매몰을 100% 방지.
* **실증 대상지 및 그라운드 트루스**: 
  * 실제 민원이 빈발하는 **서울특별시 용산구**를 실증 대상지로 포커싱.
  * 입지 예측의 신뢰도를 증명할 그라운드 트루스로 **'용산구 관내의 실제 불법 흡연 민원 데이터(E1)'**를 연동 및 팩트 검증.

---

## 🏥 2. 보건소 소관 모듈식 인프라 확장 구조 (Plug & Play)
단일 부서(지자체 보건소)에서 예산 일괄 편성 및 수의계약 발주가 가능하게 하여 B2G 사업성을 극대화했습니다.

1. **[MVP] 스마트 흡연부스 및 금연유도 CCTV 모듈**:
   * 버스/지하철 유동 거점 수요, 어린이집(30m)/학교(200m) 법적 절대 규제선 회피, E1 불법흡연 민원 가점 연동.
2. **[확장 1] 폭염/한파 실버 스마트 쉼터 모듈**:
   * 고령층 인구 밀집 주거지 수요, 혹서기/혹한기 기후 취약 어르신 동선 분석, 노인복지 조례 RAG 심의.
3. **[확장 2] 24시간 실외 스마트 AED(자동심장충격기) 응급구호 부스 모듈**:
   * 야간/공휴일 폐쇄 한계를 가진 건물 내부 AED의 사각지대 극복, **전국 2만 8,000대 KT 유휴 공중전화 부스 업사이클링 연계**, 응급의료법 RAG 심의.
4. **[확장 3] 지상 화재안전 특화 EV 충전허브 모듈**:
   * 전기차 지상 충전 시설 지상 이전 수요 및 119안전센터 골든타임 도달 권역(ST_Distance) 연계 입지 분석.

## 🔄 3. 5단계 지능형 입지 선정 & 갈등 예측 파이프라인 (E2E Flow)
지자체 폐쇄망(망분리) 환경을 완벽하게 만족시키기 위해 외부 실시간 API 호출을 배제하고 **오프라인 CSV 파일 임포트 방식**으로 파이프라인을 구축했습니다.

```
[1단계] 데이터셋 업로드 ➔ [2단계] AI 감리/HITL ➔ [3단계] AHP & PostGIS 필터링 
                                                             ▼
[5단계] GPT-4o 토론 스트리밍 ⬅ [4단계] 지역 갈등 민감도 벡터 (Top 1~3 UX) 주입
```

1. **1단계 (일괄 업로드):** 규제 조례 및 정형/공간 데이터셋을 단 한 번의 드래그앤드롭으로 일괄 투척.
2. **2단계 (AI 감리 및 HITL):** 백엔드 자동 라우팅, 결측치 자동 지오코딩 및 지도 상의 시각적 핀 드래그 보정(**Visual HITL**).
3. **3단계 (AHP Lock & PostGIS):** 일관성 비율(C.R. < 0.1) 가중치 Lock 및 법규 금지구역 차집합(`ST_Difference`) 처리로 최종 후보지 도출.
4. **4단계 (갈등 민감도 벡터 산출):** 민원수와 연령 인구를 결합해 후보지별 민감도 점수($CSS(p)$) 산출 및 토론 통제 변수로 주입. Top 2 & 3은 선택적 실행 버튼 배치로 API 비용 최적화.
5. **5단계 (AI 3대 시나리오 예측):** 찬반 주민/공무원 가상 토론을 통해 [일반/우호/불합의] 예측 시나리오 도출 및 실시간 SSE 스트리밍 중계.
*   *(선택적 피드백 루프)* **사후 Audit AI 매핑:** 실제 고문서 PDF를 검증하여 RAG `[실제사례]` 세그먼트에 분리 적재 (피드백 오염 방지).

## 📶 4. KT 그룹사 비즈니스 및 기술 시너지 (ARR & Lock-in)
* **KT 직접 고정 반복 매출 (ARR)**: 
  * 지자체 보건소의 분석 신뢰성 가점을 위한 **KT 생활인구 API 연간 라이선스 구독료 수입**.
  * 시공된 스마트 부스마다 원격 관제 및 안심벨 가동을 위한 KT LTE-M/NB-IoT 회선 월정액 납부.
* **KT 그룹사 기술 인프라 락인**: 
  * 공공 보안성 심의 통과를 위해 플랫폼의 API 및 공간 DB 서버 배포지를 **KT Cloud 공공 Zone**으로 표준 지정.
  * 정보 외부 유출을 차단하기 위해 **KT의 한국어 특화 초거대 AI '믿음(Mi:dm)' sLLM**을 온프레미스 형태로 로컬 가동.
* **KT링커스 옥외광고(DOOH) 및 충전망 전환**: 
  * 전국 유휴 공중전화 부스를 개조 시 **KT Digico 디지털 사이니지**를 결합하여 B2B/B2G 광고 수익을 쉐어, 18개월 내 초기 시공비 전액 회수.

## 🚧 5. R&D 한계점 및 향후 고도화 로드맵
* **[공간 정보] 2D 평면 공간 분석의 한계**: 향후 국토부 **3D 브이월드(Vworld) 건물 모델 및 디지털 트윈**과 연계해 기류 시뮬레이션(CFD) 기반 실외 공기질/연기 수직 확산 및 빌딩풍 물리 분석 탑재 예정.
* **[인공 지능] 외부 LLM 의존 지연**: 공공 행정 문서 및 조례 특화 sLLM 자체 파인튜닝을 통해 외부 API 단가를 0원으로 낮추고 심의 레이턴시를 **5초 이내**로 단축.
* **[데이터셋] 정적 CSV 의존 한계**: **국민신문고 API 및 KT IoT 비상벨 호출 신호**와 Websocket/Kafka 스트림 실시간 연동을 통한 도시 라이브 모니터링 시스템 구축.

## 📅 6. 8주 개발 WBS 및 주니어 R&R (A~G)
*   **주니어 A (GIS/지도):** Mapbox GL JS 로컬 렌더링, GeoJSON 격리 레이아웃 시각화 및 TOP 3 말풍선 동적 시각화 전담.
*   **주니어 B (FE/UI):** Next.js 웹 프레임워크 구축, BFF API 라우팅, 드래그앤드롭 업로드 UX 및 SSE 스트리밍 컴포넌트 개발.
*   **주니어 C (데이터):** 19종 데이터 정제, `EPSG:4326` 좌표계 투영, PostGIS 차집합 연산(`ST_Difference`) 및 버퍼 공간 쿼리 최적화.
*   **주니어 D (BE/백엔드):** FastAPI-SQLAlchemy Monolith 코어 구축, AHP C.R. < 0.1 검증 로직 및 `ahp_models` DB 락(Lock) 구현.
*   **주니어 E (AI):** AI 감리 엔진, LangGraph RAG 3자 모의 토론 프롬프트 및 Audit AI 공문서 팩트체크 파이프라인 개발 (GPT-4o 제어).
*   **주니어 F (산출물):** HTML-to-PDF 컴파일러 연동, WeasyPrint 보고서 템플릿 마크업 및 도로점용료 자동 산식 계산 바인딩.
*   **주니어 G (Infra/DevOps):** Docker Compose 환경 셋업, AWS EKS Namespace 다중 테넌트 격리 및 KMS 보안 통제.
