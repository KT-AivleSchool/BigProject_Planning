# D-2 부지 스크리닝 모듈 사용 명세서

## 0. 목적
SDSS 모델이 산출한 **추천 입지 후보 좌표**가 실제로 흡연부스를 놓을 수 있는
**공공성 부지(도로·공원·주차장 등)인지**를, 연속지적도의 **지목**으로 1차 판정한다.

> ⚠️ 한계 선언: 연속지적도에는 **소유구분(국유/공유/사유)이 없다.**
> 따라서 본 모듈의 출력은 "소유 확정"이 아니라 **지목 기반 점용 가능성 스크리닝(근사)** 이다.
> 최종 점용 가부는 브이월드 토지특성/국공유재산 데이터로 별도 확인해야 한다.

---

## 1. 준비 데이터 (입력)

### 1-1. 추천 입지 후보 (SDSS 산출물, 추후 연결)
- 형식: CSV, UTF-8(-sig)
- **필수 컬럼**: `후보지명`, `경도`(lng), `위도`(lat)
- 좌표계: EPSG:4326 (일반 GPS 위경도)
- 예시:

| 후보지명 | 경도 | 위도 |
|---|---|---|
| 삼각지역 3번출구 앞 | 126.9726 | 37.5341 |
| 용산역 광장 | 126.9646 | 37.5299 |

*(현재는 임시 형식. SDSS 모델 완성 시 이 스키마로 좌표를 내보내면 그대로 연결됨)*

### 1-2. 연속지적도 SHP (D-2, 이미 확보)
- 파일: `AL_D002_11_YYYYMMDD.shp` (+ `.dbf .shx .prj .cpg` 세트 — 같은 폴더 유지 필수)
- 범위: 서울특별시 전체(코드 11), 약 90만 필지, POLYGON
- 좌표계: **EPSG:5186** (GRS80)
- 속성 컬럼: A0(일련번호) / **A1=PNU** / A2(법정동코드) / **A3=법정동명** / **A4=지번** / **A5=지번+지목(끝글자=지목부호)** / A6(기준일) / A7(시군구코드) / geometry

### 1-3. 실행 환경
- Python + `geopandas`, `shapely`, `pandas`
- **SHP 읽기 엔진 필요**: geopandas 1.x는 `pyogrio` 사용 → 없으면 `pip install pyogrio`
- ⚠️ Python 3.14에서 설치가 막히면 → **3.11 가상환경** 또는 **QGIS**로 대체(§6)

## 2. 처리 흐름 (파이프라인)

```
[추천 좌표 CSV]                    [연속지적도 SHP]
  경도/위도(4326)                   필지 폴리곤(5186)
      │                                  │
      ▼                                  │
  Point 생성(4326)                       │
      │                                  │
      ▼  좌표계 통일                      │
  4326 → 5186 변환 ────────────┐         │
      │                        ▼         ▼
      └──────────────► 공간조인(sjoin, within)
                              │
              within 실패 시  ▼  (경계 걸침/도로중앙 보정)
                    최근접 필지(20m 이내)
                              │
                              ▼
                   A5 끝글자 → 지목 추출
                              │
                              ▼
                   3단계 등급 매핑(§3)
                              │
                              ▼
                 [결과 CSV: 후보별 지목·등급·PNU]
```

단계 요약: ①좌표 로드→Point(4326) ②지적도 crs(5186)로 변환 ③`sjoin(predicate="within")`
④within 실패분은 **20m 이내 최근접 필지**로 보정(초과=미매칭) ⑤A5 끝글자로 지목 판별
⑥3단계 등급 매핑 ⑦저장.

---

## 3. 지목 판정 기준 (3단계) — 조정 가능

> ⚠️ **지목 부호 정정**: 흔한 오류로 "주차장=주, 공장용지=공"으로 쓰는 경우가 있으나 틀림.
> 정확히는 **주차장=`차`**, **주유소용지=`주`**, **공원=`공`**, **공장용지=`장`**.

| 등급 | 의미 | 지목(부호) |
|---|---|---|
| **점용후보** | 공공성 높음·설치 현실적 | 도로(**도**), 공원(**공**), 주차장(**차**) |
| **조건부** | 공공일 수 있으나 애매·현장확인 필요 | 하천(천), 구거(구), 유지(유), 제방(제), 체육용지(체), 유원지(원), 철도용지(철), 수도용지(수), 잡종지(잡) |
| **보류** | 사유지 가능성 높음·제외 | 대·전·답·과·목·임·광·염·공장용지(장)·학교용지(학)·주유소용지(주)·창고(창)·양어장(양)·종교(종)·사적(사)·묘지(묘) |

- 현재 기준: **Q1 "현실적 넓게"**(점용후보=도로+공원+주차장) + **Q2 3단계**.
- 코드 상단 `GRADE_점용후보`, `GRADE_조건부` 상수로 노출 → 잡종지 상·하향 등 즉시 조정 가능.
- "광장"은 독립 지목 아님(도시계획시설) → 지목상 대개 '도로'로 잡힘.

---

## 4. 출력 예상물

파일: `좌표별_부지판정.csv` (UTF-8-sig)

| 컬럼 | 설명 |
|---|---|
| 후보지명 | 입력 그대로 |
| 경도, 위도 | 입력 좌표(4326) |
| PNU | 매칭 필지 고유번호(A1) |
| 법정동명 | A3 |
| 지번 | A4 |
| 지목 | A5 끝글자 판별(예: 도로/공원/대) |
| **판정등급** | 점용후보 / 조건부 / 보류 / 미매칭 |
| 매칭방식 | within / nearest(보정) / 미매칭 |

콘솔 로그: 등급 분포, 매칭방식 분포.

## 5. 실행 방법 (쓰는 법)

### 5-1. 준비
1. 연속지적도 압축 해제 → **세트파일(.shp .dbf .shx .prj .cpg)을 같은 폴더**에 둔다.
2. 입력 좌표 CSV(`추천입지_후보.csv`)를 `후보지명,경도,위도` 컬럼으로 준비한다.
3. (엔진 없을 때) `pip install pyogrio`

### 5-2. 실행
```cmd
python screen_land_use.py
```
→ 같은 폴더에 `좌표별_부지판정.csv` 생성, 콘솔에 등급/매칭 분포 출력.

### 5-3. 조정 포인트 (스크립트 상단 상수)
| 상수 | 기본값 | 용도 |
|---|---|---|
| `LAND_SHP` | AL_D002_11_20260608.shp | 연속지적도 파일명 |
| `COORD_CSV` | 추천입지_후보.csv | 입력 좌표 파일명 |
| `NAME_COL/LNG_COL/LAT_COL` | 후보지명/경도/위도 | 입력 컬럼명 |
| `NEAREST_MAX_M` | 20 | within 실패 시 최근접 보정 허용거리(m) |
| `GRADE_점용후보` / `GRADE_조건부` | §3 표 | 지목 등급 집합 |

### 5-4. 임시 테스트 팁 (SDSS 좌표 나오기 전)
- 이미 확보한 용산 좌표로 미리 검증 가능:
  - **A-1 무단투기 지오코딩 결과**(`..._geocoded.csv`)를 `후보지명,경도,위도`로만 재구성해 입력 → 실제 단속지점이 어떤 지목에 떨어지는지 확인.
  - 또는 버스정류소/역사마스터 용산 좌표를 넣어 도로·역사 필지 판정 감을 잡는다.

### 5-5. 결과 해석
- **점용후보**만 곧바로 확정하지 말 것 — 지목이 '도로'여도 사유도로 가능 → §6 소유구분 보강 후 최종.
- **nearest / 미매칭**이 많으면 입력 좌표 정밀도(도로중앙 vs 보도) 재점검.

## 6. 한계·주의 & 후속 보강
**한계**
- **소유구분 없음** → 지목 '대'라도 국유 가능, '도로'여도 사유 가능. **확정 아님.**
- 경계선 정확히 걸치면 within 실패 → nearest 보정(플래그로 구분).
- 도로 **중앙 좌표**는 '도로' 필지로 과대추정 가능.
- 본 지적도는 **법적 측량·소유권 확인용 아님**(참조용 도면).

**후속 보강(권장)**
- **소유구분 확정**: 브이월드 **토지특성정보 API** 또는 국공유재산 데이터를 PNU로 결합.
- **면적·접도 조건**: 필지 면적·도로 접함을 폴리곤에서 계산해 최소면적 필터 추가.
- **설치 대안(3.14 이슈 시)**: `py -3.11 -m venv gis_env` 가상환경, 또는 QGIS '위치로 조인'.
