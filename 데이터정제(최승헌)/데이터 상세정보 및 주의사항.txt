그룹 A (공공데이터포털, 로그인 없이 즉시): ⑬무단투기·⑦상가·⑭보호구역
그룹 B (바로가기→서울열린데이터): ①금연구역·②어린이집
그룹 C (서울열린데이터 파일, 로그인 없이): ③좌표매개표 2종·④승하차 2종·⑤생활인구 — 월별은 최근 1개월만 골라받기
그룹 D (브이월드 회원가입 후 SHP): ⑥경계·⑨연속지적
그룹 E (활용신청+인증키): ⑧민원 API·브이월드 지오코딩

용산구 시군구 코드: 11170 (통계/주소 필터 기준값)

API 키 필요 데이터 
-지오코드: A1(담배꽁초 상습무단투기지역현황), 
-Data.go.kr API인증키 : E1(민원데이터 조회)

1. 민원 데이터는 위치 정보가 안나온다면 그냥 해당 구 선택 지표로 사용 (민원수가 많은 구에 필요한 흡연부스)
 <- 꼭 흡연이 아니더라도 관련 민원이 많으면 그 민원에 해당하는 서비스 수요 확인가능

 2. 상업적 이용이 금지된 데이터는 (제 4유형) 확인 필요함 (B2B의 경우)
현재 ②어린이집·⑥경계·⑧민원 3개가 상업이용금지

 3. 서울시 금연구역 - 구로, 강북, 양천구 밖에 없음 -> 대신 용산구 흡연구역 데이터 확보, 금연주택 아파트 단지

4. 버스 정류소 위치 역 지오코딩으로 용산구 근처 추정 데이터만 뽑음
5. 역 지오코딩시 좌표기준으로 판단

6. C3 버스정류소 승하차인원 데이터 필터링시 C1 코드에서 나온 버스정류소_ARS목록 데이터를 같이 사용
- 버스  정류소 승하차 인원데이터는 1개월 말고 3~12개월 평균 데이터를 사용하는게 더 정확할것 같음

7. 마찬가지로 C4 지하철역별 승하차인원도 C2의 용산구 지하철역명으로 필터링해 사용

8. 민원 데이터 2025는 처리기관별이라 민원 발생지별 2024 API로 처리
