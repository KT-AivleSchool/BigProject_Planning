## 📢 조장 단독 개발 일일 공유 및 전파사항 (2026-07-06 - 2차 고도화 스프린트 완료)

본 문서는 RAG 조례 독립 모달 분리 및 중복 방지, 개별 물리 파일/캐시 소거 기능을 통합 구현 및 검증한 후 작성한 추가 일지입니다.

### ✅ 1. 금일 R&D 성공 사항 (Success Logs)
1.  **조례 PDF 독립 관리 창구 및 다중 업로드(Multi-Upload) 구현**:
    *   메인 Left Panel의 복잡성을 낮추기 위해 조례 PDF 업로더를 상단 헤더의 전용 모달 창구로 격리 분리했습니다.
    *   `POST /api/v1/upload/regulation` API 및 프론트 `handleRegulationFileChange`를 개편하여 한 번에 여러 개의 법규 PDF 파일이 등록 가능하도록 다중 업로드 규격 구현 완료.
2.  **📋 조례 목록 조회 및 독립 모달 UI 적용**:
    *   상단 헤더에 `📋 조례 목록 조회` 버튼을 신설하여, 백엔드 `GET /api/v1/upload/regulations` API와 통신해 등록된 조례(파일명 및 용량)를 모달 내에 리스팅 완료.
3.  **조례 중복 차단 (Duplication Guard)**:
    *   동일 파일명의 PDF가 다시 업로드될 경우 백엔드에서 `400 Bad Request` 에러를 반환하여 RAG 인프라의 텍스트 분석 중복 오류를 방지하고 프론트에 경고를 표출하도록 예방 조치했습니다.
4.  **조례 물리 파일 및 텍스트 캐시 수거 (Deletion Engine)**:
    *   `DELETE /api/v1/upload/regulations/{filename}` API를 구현하여, 파일 삭제 시 원천 저장소의 PDF뿐 아니라 RAG 분석 용도로 생성된 `[filename].txt` 텍스트 캐시 파일까지 디렉토리에서 동시 제거해 용량 누수를 차단했습니다.
    *   조례 목록 모달 내에 휴지통(`🗑️`) 버튼을 바인딩하여 정상 동작을 검증했습니다.
5.  **통합 검증 및 Next.js 빌드 성공**:
    *   `e2e_test.py` 스크립트를 통한 목록-삭제-신규등록-중복차단 시나리오 E2E 자동화 검증 완료 (전부 200 OK 및 400 Bad Request 획득).
    *   `npm run build` 컴파일 통과 및 Leaflet GIS 동결 코드 안정성 확보.

### ❌ 2. 트러블슈팅 및 버그 픽스 (Failure Logs)
1.  **PowerShell 파이프라인 문자열 따옴표 파싱 에러**: E2E 스크립트 실행 시 띄어쓰기 경로 처리 오류 ➔ `&` 호출 접두사 바인딩으로 해결.
2.  **Next.js API 프록시 ECONNRESET 좀비 포트 충돌**: 백엔드 포트 행업 에러 ➔ 백엔드 uvicorn을 python venv 모듈 기반으로 재기동하여 정상 처리.
3.  **회원가입/JWT 로그인 명세 불일치**: 1차 프로토타입 범위에 미구현된 회원가입 요소를 요구사항정의서에서 2차 EKS 격리 단계로 이관 처리하고 프론트 모킹으로 고정.
