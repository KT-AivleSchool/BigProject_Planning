# 📱 [제출용] OmniSite 4대 화면 UI/UX 요약 설계서 (v1.0.0-prototype)

본 문서는 **"9기 빅프로젝트 산출물 워크시트(AI트랙)"**의 UI/UX 설계서 최종 제출 포맷(6대 컬럼 명세)에 맞추어 작성된 보고서입니다. 
로컬에 수록된 4종의 와이어프레임 설계 이미지(`omnisite_uiux_sketch_oms01.png` ~ `omnisite_uiux_sketch_oms04.png`)와 100% 매핑됩니다.

## 🖥️ 1. 메인 지도 기반 의사결정 대시보드 (OMS-01-01-001)

*   **화면 ID:** `OMS-01-01-001`
*   **화면 경로:** 홈 > 입지분석 대시보드
*   **화면명:** 메인 대시보드 화면
*   **서버:** WEB (Next.js Frontend)
*   **버전:** 1.0

### 화면 설명
*   **[1-1] Mapbox GIS 메인 지도 영역:** PostGIS 공간 연산이 처리된 실시간 오버레이 레이어가 렌더링되는 지도 뷰어 영역입니다. `ST_Union` 기반 법적 배제 구역은 반투명 적색 구역(Red Zone) 마스크로 고정 표시되며, AHP 연산 결과 도출된 최적 입지 3곳이 녹색 후보 필지 및 핀 마커로 시각화됩니다.
*   **[1-2] 데이터 일괄 업로드 드롭존 패널:** 공간/통계 파일(SHP, CSV) 및 행정 조례 파일(PDF, HWP)을 한 번의 드래그앤드롭으로 일괄 수신하는 업로드 영역입니다. 파일별 파싱 진행률 및 결측 데이터 감리 상태를 노출합니다.
*   **[1-3] AHP 가중치 설정 슬라이더:** 유동인구, 민원 밀도, 배후지 등 5대 입지 인자 간의 상대적 가중치를 1~9점 척도 쌍대비교 슬라이더로 조절하는 인터페이스입니다. 실시간 일관성 비율(C.R.)을 연산하여 C.R. < 0.1 검증 만족 시 가중치를 영구 잠금(Lock)하여 저장합니다.
*   **[1-4] 후보지 세부 통계 정보 카드:** 지도 상의 핀을 클릭했을 때 활성화되는 세부 정보창입니다. 선택 필지의 PNU, 지목, 소유구분(국유지 등), 공시지가 상세 스펙과 주변 배후 민원수 기반의 $CSS(p)$ 지역 갈등 민감도 등급(상/중/하) 및 지수 그래프를 보여줍니다.

## 🖥️ 2. 데이터 적재 및 자동 감리 화면 (OMS-01-02-001)

*   **화면 ID:** `OMS-01-02-001`
*   **화면 경로:** 홈 > 데이터 적재 관리
*   **화면명:** 데이터 업로드 및 감리 화면
*   **서버:** WEB (Next.js Frontend)
*   **버전:** 1.0

### 화면 설명
*   **[2-1] 파일 일괄 업로드 드롭존:** 사용자가 공간 데이터(SHP)와 정형 수치 데이터(CSV), 조례 문서(PDF)를 업로드하는 물리적 입력 영역입니다. 드롭 즉시 로딩 프로그레스 바가 출력되며 비동기 파싱이 구동됩니다.
*   **[2-2] 자동 지오코딩 및 API 상태 모니터:** 주소 정보만 있고 위경도가 없는 CSV 파일 수신 시 백엔드의 Kakao/Vworld API 지오코딩 변환율(예: 1205/1500건)과 실패 행(Row) 주소 리스트를 붉은 경고창에 표출하는 모니터링 컴포넌트입니다.
*   **[2-3] 컬럼 시맨틱 매핑 및 스키마 검증 UI:** 업로드 데이터의 컬럼명이 상이할 경우, AI 스키마 감리 판정 결과에 따라 사용자가 드롭다운 메뉴를 조작하여 위도/경도/PNU 등 표준 스키마 필드에 수동 매핑하는 보정 인터페이스입니다.
*   **[2-4] 임시 공간 데이터 HITL 검증 맵:** 지오코딩이 적용된 좌표 핀들이 CAD/GIS 지형도 상에 임시 오버레이되는 미니 맵입니다. 공무원이 지오코딩 오류 핀을 직접 마우스 드래그로 수동 교정(Drag to Correct)하고 확정 등록할 수 있습니다.

## 🖥️ 3. AI 모의 심의 토론 및 시뮬레이션 콘솔 (OMS-01-03-001)

*   **화면 ID:** `OMS-01-03-001`
*   **화면 경로:** 홈 > 입지분석 > 갈등 시뮬레이터
*   **화면명:** 갈등 예측 시뮬레이터 화면
*   **서버:** WEB (Next.js Frontend)
*   **버전:** 1.0

### 화면 설명
*   **[3-1] AI 에이전트 실시간 모의 토론 터미널:** 찬반 주민대표 및 구청 공무원 페르소나를 탑재한 에이전트들이 RAG 조례 텍스트와 지역 갈등 민감도 벡터를 기반으로 심의 논쟁을 벌이는 콘솔 창입니다. FastAPI SSE 스트리밍 연동으로 브라우저에 대사가 한 자씩 실시간 타이핑 렌더링 처리됩니다.
*   **[3-2] 3대 스펙트럼 예측 시나리오 카드:** 가상 모의 심의를 통해 추론된 3가지 타결 시나리오(일반적 협상 / 우호적 타결 / 교착 상태) 카드로, 각 분기별 예측 확률 및 통계적 신뢰도 점수를 게이지바로 노출합니다.
*   **[3-3] WeasyPrint PDF 보고서 출력 & 사후 Audit 피드백 바:** 심의가 완료된 타당성 리포트(HTML 바인딩)를 정식 인쇄용 PDF 규격으로 내려받는 컴포넌트입니다. 우측 하단에는 실제 설치 이후 공문 PDF를 드래그앤드롭하여 업로드하는 사후 팩트체크 검증존이 제공됩니다.

## 🖥️ 4. 의사결정 이력 및 사후 검증 대시보드 (OMS-01-04-001)

*   **화면 ID:** `OMS-01-04-001`
*   **화면 경로:** 홈 > 의사결정 관리 대시보드
*   **화면명:** 이력 조회 및 사후 검증 화면
*   **서버:** WEB (Next.js Frontend)
*   **버전:** 1.0

### 화면 설명
*   **[4-1] 의사결정 이력 마스터 테이블:** 과거에 수행한 날짜, 자치구역, 분석 인프라 종류별 Top 1~3 부지 PNU를 표(Table) 형태로 열람하고 행정 처분 종결 여부를 확인하는 필터 테이블입니다.
*   **[4-2] 실재 이행 공문서(PDF) 업로드 폼:** 실제 현장에 설치가 완료되었음을 증빙하는 기안 결재서 혹은 조례 고시 공문 PDF 파일을 마우스로 끌어다 적재하는 업로드 양식 영역입니다.
*   **[4-3] Audit AI 매핑 판정 UI:** 업로드된 공문서를 OCR 분석하여 추출한 팩트 텍스트와 설계 단계의 예측 시나리오 간 코사인 유사도를 연산하여, 실제 도달 결과를 매핑 표시하는 검토 인터페이스입니다.
*   **[4-4] RAG 독립 사례 적재 이력:** 팩트체크 검증이 종결된 데이터를 RAG 격리 영역(`verified_precedents` 테이블)에 적재한 누적 데이터양과 요약된 통계를 조회하는 모니터링 영역입니다.
